# 🏷️ Kobo Series Metadata  
This document explains how to ensure **series metadata** (series name + series index) displays correctly on Kobo e‑readers. Kobo devices do **not** read standard EPUB metadata, so additional steps are required.

This guide covers:

- Why Kobo ignores standard EPUB series metadata  
- How Calibre fixes it  
- How KoboTouchExtended injects Kobo‑specific metadata  
- How to configure Calibre correctly  
- How to verify metadata on the device  
- How to fix broken metadata retroactively  

---

# ❗ Why Kobo Ignores Standard EPUB Series Metadata

EPUB files typically store series metadata using:

```
calibre:series
calibre:series_index
```

However, Kobo’s firmware **does not read these fields**.

Instead, Kobo expects:

- A proprietary metadata block  
- Injected into the EPUB/KEPUB  
- Using Kobo‑specific tags  
- With a specific internal structure  

If these tags are missing, Kobo will show:

- No series name  
- No series index  
- Incorrect sorting  
- Incorrect metadata  

This is why series metadata often appears broken on Kobo devices.

---

# ✔ How Calibre Fixes Series Metadata

Calibre includes a device driver called **KoboTouchExtended** (KTE).  
This driver:

- Injects Kobo‑specific metadata into EPUB/KEPUB files  
- Ensures Kobo recognizes series name + index  
- Updates metadata on the device  
- Updates the device’s SQLite database  
- Generates thumbnails  
- Sends KEPUBs instead of EPUBs (optional but recommended)

This is the correct way to fix series metadata for Kobo.

---

# 🛠 Enabling Series Metadata Fixes in Calibre

### 1. Open Calibre  
Go to:

```Preferences → Plugins → Device Interface```

### 2. Select **KoboTouchExtended**  
Click:

```Customize plugin```

### 3. Enable the following options:

[✔] Set series information

[✔] Set series index

[✔] Upload covers

[✔] Generate thumbnails

[✔] Use KEPUB format


### 4. Restart Calibre  
This ensures the driver loads correctly.

---

# 🔄 How Metadata Is Applied to the Device

When you send a book to your Kobo (via USB):

1. CalibreTouchExtended injects Kobo‑specific metadata  
2. The device receives a modified EPUB/KEPUB  
3. Kobo updates its internal database (`KoboReader.sqlite`)  
4. Kobo displays:
   - Series name  
   - Series index  
   - Correct sorting  
   - Correct cover  

This works for:

- EPUB → KEPUB conversions  
- KEPUBs generated by kepubify  
- Books imported from LazyLibrarian  
- Books manually added to Calibre  

---

# 🧪 Verifying Series Metadata on the Device

On your Kobo:

1. Open the book’s **Details** page  
2. You should see:

```
Series: <Series Name>
Book <Index> of <Series>
```

3. In the library view, sorting by **Series** should work correctly  
4. Shelves should group series correctly (if enabled)

If metadata is missing:

- The book may not have been processed by KTE  
- The book may not have been resent after metadata changes  
- The device database may need a refresh (see below)

---

# 🛠 Fixing Broken Series Metadata (Retroactively)

If a book already exists on the device with incorrect metadata:

### Option A — Resend the book  
1. Delete the book from the device  
2. In Calibre, right‑click → “Send to device”  
3. Eject the device  
4. Kobo will rebuild metadata

### Option B — Force metadata update  
1. Connect Kobo  
2. In Calibre, select the book  
3. Click:

```Device → Update metadata on device```

### Option C — Rebuild the device database  
If metadata is severely corrupted:

1. Connect Kobo  
2. Backup:

```.kobo/KoboReader.sqlite```

3. Delete the file  
4. Eject the device  
5. Kobo will rebuild the database automatically  
6. Reconnect and resend books via Calibre  

This resolves 99% of metadata issues.
