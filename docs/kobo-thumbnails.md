# 🖼️ Kobo Cover Thumbnails  
This document explains how to ensure **high‑quality cover thumbnails** appear correctly on Kobo e‑readers. Kobo devices store thumbnails separately from book files, and incorrect or missing thumbnails are one of the most common Kobo issues.

This guide covers:

- Why Kobo thumbnails break  
- How Calibre generates proper thumbnails  
- How KoboTouchExtended fixes cover issues  
- How to configure Calibre correctly  
- How to verify thumbnails on the device  
- How to repair broken thumbnails retroactively  

---

# ❗ Why Kobo Thumbnails Break

Kobo devices do **not** extract thumbnails from EPUB/KEPUB files.

Instead, they expect thumbnails to be pre‑generated and stored in:

```.kobo/images/```

Each thumbnail is:

- A JPEG file  
- Named using a hash of the book’s internal ID  
- Sized according to Kobo’s UI requirements  

If thumbnails are missing or incorrect, Kobo will show:

- Blurry covers  
- Wrong covers  
- Placeholder covers  
- Covers that never update  
- Covers that disappear after a reboot  

This is why proper thumbnail generation is essential.

---

# ✔ How Calibre Fixes Thumbnails

Calibre’s **KoboTouchExtended** (KTE) driver:

- Generates high‑quality thumbnails  
- Places them in `.kobo/images/`  
- Ensures the correct hash naming  
- Updates thumbnails when metadata changes  
- Sends thumbnails along with the book  

This is the correct way to ensure Kobo displays covers properly.

---

# 🛠 Enabling Thumbnail Generation in Calibre

### 1. Open Calibre  
Go to:

```Preferences → Plugins → Device Interface```

### 2. Select **KoboTouchExtended**  
Click:

```Customize plugin```

### 3. Enable the following options:

[✔] Upload covers

[✔] Generate thumbnails

[✔] Set series information

[✔] Set series index

[✔] Use KEPUB format


### 4. Restart Calibre  
This ensures the driver loads correctly.

---

# 🔄 How Thumbnails Are Applied to the Device

When you send a book to your Kobo (via USB):

1. CalibreTouchExtended generates a thumbnail  
2. The thumbnail is saved to:  
```.kobo/images/<hash>.jpg```
3. Kobo updates its internal database (`KoboReader.sqlite`)  
4. The library view displays the correct cover  

This works for:

- EPUB → KEPUB conversions  
- KEPUBs generated by kepubify  
- Books imported from LazyLibrarian  
- Books manually added to Calibre  

---

# 🧪 Verifying Thumbnails on the Device

On your Kobo:

1. Open the **Library** view  
2. Covers should appear crisp and correct  
3. Sorting by **Recent** should show updated covers  
4. Opening the book’s **Details** page should show the correct cover  

If covers are missing or incorrect:

- The thumbnail may not have been generated  
- The device database may need a refresh  
- The book may need to be resent  

See the troubleshooting section below.

---

# 🛠 Fixing Broken Thumbnails (Retroactively)

If a book already exists on the device with incorrect or missing thumbnails:

### Option A — Resend the book  
1. Delete the book from the device  
2. In Calibre, right‑click → “Send to device”  
3. Eject the device  
4. Kobo will rebuild metadata and use the new thumbnail  

### Option B — Force metadata + cover update  
1. Connect Kobo  
2. In Calibre, select the book  
3. Click:

```Device → Update metadata on device```

### Option C — Rebuild the device database  
If thumbnails are severely corrupted:

1. Connect Kobo  
2. Backup:

```.kobo/KoboReader.sqlite```


3. Delete the file  
4. Eject the device  
5. Kobo will rebuild the database automatically  
6. Reconnect and resend books via Calibre  

This resolves 99% of thumbnail issues.
